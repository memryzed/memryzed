# Development and operations

This directory contains the documentation for contributors, maintainers,
and operators of Memryzed. It covers how to set up the development
environment, how the project is organized for change management, how
releases are cut, and how incidents are handled.

If you are a user of Memryzed, you do not need any of these documents.
See `../README.md` for the user-facing index.

## Contents

- `setup.md` - getting a working development environment.
- `branching.md` - git workflow, branch naming, and pull requests.
- `testing.md` - how tests are organized and run.
- `release-process.md` - cutting a release, including patch and
  security releases.
- `changelog-conventions.md` - how to write `CHANGELOG.md` entries.
- `versioning.md` - the project's semantic versioning policy.
- `incident-response.md` - what to do when something breaks for
  users.

## When to read what

Before your first contribution: `setup.md`, `branching.md`,
`testing.md`, `changelog-conventions.md`.

Before your first release: `release-process.md`, `versioning.md`.

When something is wrong in production: `incident-response.md`.
