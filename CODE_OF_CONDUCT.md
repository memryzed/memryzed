# Code of Conduct

## Purpose

Memryzed is built by and for software developers. This code of conduct
sets the expectation for how we work together, so that the project remains
a place where people can do good work without dealing with unprofessional
behavior.

## Expected behavior

- Be respectful in technical disagreements. Critique ideas, not people.
- Assume good faith. Most disagreements are misunderstandings.
- Keep discussions focused on the project. Off-topic political,
  religious, or personal commentary belongs elsewhere.
- Give credit where it is due. Acknowledge prior work and other
  contributors.
- Accept that not every contribution will be accepted. Maintainers
  decide what fits the project.

## Unacceptable behavior

- Personal attacks, insults, or harassment of any kind.
- Discriminatory language or behavior, including but not limited to
  language directed at race, ethnicity, religion, gender, sexual
  orientation, disability, or national origin.
- Sustained disruption of discussion, issue trackers, or pull requests.
- Publishing other people's private information without consent.
- Sexualized language or imagery.

## Scope

This code of conduct applies to all project spaces: the GitHub
repository, the issue tracker, pull requests, project chat channels if
they exist, and project events. It also applies to representations of
the project in other spaces.

## Reporting

If you see behavior that violates this code of conduct, report it to
`conduct@memryzed.com`. Reports are read by the project maintainers and
will be handled confidentially.

Reports should include:

- A description of what happened.
- Where it happened.
- Who was involved.
- Any relevant links or screenshots.

You will receive an acknowledgment within seven days.

## Enforcement

Maintainers may take any action they consider appropriate, including
warning the offender, removing comments or contributions, temporarily or
permanently banning the offender from project spaces, or escalating to
the relevant platform administrators.

Maintainers are also bound by this code of conduct. If a maintainer
violates it, the same reporting process applies.

## Attribution

This code of conduct is informed by the Contributor Covenant
(https://www.contributor-covenant.org/) and adapted to the needs of this
project.
